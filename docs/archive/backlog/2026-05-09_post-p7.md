## B2 — `_executor_helpers.py` coverage 76% (Java compile 분기)

### 위치
`ipe/nodes/_executor_helpers.py:64-67, 84-96, 103-105`

### 문제
P6 audit에서 분리한 후 처음 측정된 coverage 76%. 미커버 라인은 모두 Java
`_compile`/`_run_cmd`의 javac 분기. 테스트 환경에 `javac` binary가 없어
실측 검증 불가.

### 미뤄둔 이유
- `_compile`은 Python 분기(`return True, ""`)만 활성. javac 분기는 통합 테스트
  대상이지 단위 테스트 책임 아님.
- 분기 자체는 단순 — `runner.run()` 호출만. mocking으로 cover 가능하나,
  실측 가치가 없음.
- P3 audit B5(sandbox-exec process-exec)와 동일하게 dev 환경 한계.

### Resolution 시점
P12 GitHub Actions CI에서 `apt-get install default-jdk` + Java sample 솔루션 통합
테스트 추가 시 자연 해소. CI 도입과 묶음.
