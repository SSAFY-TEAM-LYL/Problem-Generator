## C2 — `tests/test_observability.py`에 ReplayTracker 단위 테스트 미작성

### 위치
`tests/test_observability.py` — `LLMCallTracker` 단위 테스트만 존재. `ReplayTracker` 미커버.

### 문제
P8.3에서 추가한 `ReplayTracker(LLMCallTracker)`는 `_load_traces` (`@staticmethod`),
`invoke` override를 가짐. 통합 테스트(`test_replay.py`)에서 happy path와
"replay miss when empty" 2 cases만 커버. 다음 edge cases 미검증:

- `_load_traces`: malformed JSON / non-int seq prefix / 알 수 없는 파일명 패턴
- `invoke`: seq overshoot (cache size > seq) / cache의 cost_usd 보존 / AIMessage wrap

`ipe/observability.py` coverage 93% — 미커버 라인 158/162-163/166-167은
모두 `_load_traces`의 try/except 경로 (ValueError / JSONDecodeError).

### 미뤄둔 이유
- 통합 테스트가 happy path를 cover.
- 단위 테스트 정비는 P9에서 generator/architect/auditor 단위 테스트 라운드와
  묶어 한 번에 처리하는 것이 효율적 (B3와 동일 카테고리).

### Resolution 시점
P9 진입 시 노드별 단위 테스트 정비 라운드 (B3, A6 통합)에서 처리.
