## 1. 동기

R5 패턴 (brute oracle Phase A) 효과 확인 후 RFC sequencing 1순위:
- **단일 LLM call 위에 결정적 cross-check layer**가 stochasticity 보완의 핵심
- ECC의 `PreToolUse` hook과 매핑 — 노드 진입 직전 mandatory check
- LLM call 비용 지불 전에 invalid state 차단 (cost ↓)

## 2. 설계 — registry + decorator + wrap

### 2.1 핵심 인프라

```python
# ipe/hooks.py

_PRE_HOOKS: dict[str, list[Callable[[ProblemState], str | None]]] = {}

def register_pre_hook(node_name: str):
    def decorator(fn):
        _PRE_HOOKS.setdefault(node_name, []).append(fn)
        return fn
    return decorator

def run_pre_hooks(node_name, state) -> str | None:
    for hook in _PRE_HOOKS.get(node_name, []):
        if reason := hook(state):
            return reason  # 첫 reject에서 short-circuit
    return None

def wrap_with_pre_hooks(node_name, fn):
    def wrapped(state):
        reason = run_pre_hooks(node_name, state)
        if reason:
            return {**state,
                    "feedback_message": f"pre-hook[{node_name}]: {reason}",
                    "last_failed_node": state.get("last_failed_node") or node_name}
        return fn(state)
    return wrapped
```

### 2.2 graph.py 통합

```python
g.add_node("coder", cast(Any,
    wrap_with_pre_hooks("coder", partial(coder.run, tracker=tracker))
))
g.add_node("executor", cast(Any,
    wrap_with_pre_hooks("executor",
        partial(executor.run, runner=runner, workdir_root=workdir_root))
))
```

cast 사용 이유: langgraph `add_node`의 NodeInputT generic이 wrap된 `Callable[[ProblemState], ProblemState]`와 직접 매칭 안 됨. `partial` 호출도 mypy가 추론 실패해 우연히 통과하던 케이스. cast로 명시.

### 2.3 Hook contract

- signature: `(state: ProblemState) -> str | None`
- return `None` → 노드 정상 진행
- return `str` → 노드 실행 skip + `last_failed_node=node_name` + feedback 설정 → `_decision` 노드가 라우팅 처리
- **state read-only** (mutation 금지) — race + immutability principle

### 2.4 Short-circuit policy

같은 node에 등록된 hook이 여러 개일 때 **첫 reject에서 중단**. 다음 hook 안 부름.
- 근거: hook이 작은 정적 check라 cumulative info 불필요. 첫 명확한 reject가 가장 actionable
- 향후 hook 수가 증가하면 priority/grouping 고려

## 3. 첫 3 builtin hook

### 3.1 `check_problem_complete` (coder)

```python
@register_pre_hook("coder")
def check_problem_complete(state) -> str | None:
    missing = [f for f in ("problem_description", "constraints", "sample_testcases")
               if not state.get(f)]
    if missing:
        return f"architect output incomplete — missing: {missing}"
    if not isinstance(state.get("sample_testcases"), list) or not state["sample_testcases"]:
        return "sample_testcases must be a non-empty list"
    return None
```

architect가 정상 응답 전에 coder 호출되는 edge case 차단. 현 graph에서는 routing이 보호하지만 hook이 명시적 contract.

### 3.2 `check_solution_code_present` (executor)

```python
@register_pre_hook("executor")
def check_solution_code_present(state) -> str | None:
    code = state.get("solution_code")
    if not code or not str(code).strip():
        return "solution_code is empty — coder must produce a solution"
    return None
```

coder가 빈 solution 반환한 edge case (예: LLM 응답이 코멘트만) 차단.

### 3.3 `check_solution_imports` (executor)

```python
_PYTHON_STDLIB = frozenset({"sys", "os", "re", "math", "collections", ...})

@register_pre_hook("executor")
def check_solution_imports(state) -> str | None:
    if state.get("target_language", "python") != "python":
        return None
    code = state.get("solution_code") or ""
    forbidden = [m.group(1).split(".")[0]
                 for m in _IMPORT_RE.finditer(str(code))
                 if m.group(1).split(".")[0] not in _PYTHON_STDLIB]
    if forbidden:
        return f"solution uses non-stdlib imports: {sorted(set(forbidden))[:5]}. " \
               f"sandbox has Python stdlib only — rewrite without these."
    return None
```

sandbox image (`python:3.11-slim`)에는 외부 패키지 없음. `import numpy` 같은 코드는 무조건 RTE → 정적 reject로 LLM call 1회 절약.

## 6. 한계 + 후속

- **Phase 전환 hook 미포함**: 첫 PR scope는 노드 진입 hook만. RFC §M2의 `pre_phase_b` (executor 내부 phase 전환) 같은 hook은 backlog. complexity feasibility check는 큰 정적 분석 → 별도 작업.
- **모든 hook fire 누적 보고 미구현**: short-circuit이라 첫 reject만. hook 수 증가 후 grouping 정책 검토.
- **e2e 효과 측정 필요**: hook이 실제 fire하는 빈도 + reject 후 다음 cycle 행동 — BFS/SegTree run으로 확인.
- **다른 노드 (architect, auditor, generator) wrap 안 함**: M1 (Coder 분해) 시 graph topology 변경하면서 함께 wrap 추가 고려.
- **builtin hook 추가 후보**: `check_constraints_structured_valid` (architect 후), `check_brute_solution_present` (R5 활용 위한 사전 확인), `check_samples_unique` (architect 출력 sanity).
