## D2 — `tests/test_evaluator_unit.py` 미작성

### 위치
`tests/test_evaluator_unit.py` — 미작성. `evaluator.py` 통합 테스트만 존재.

### 문제
P9.4의 `test_evaluator.py`는 happy + parse-fail 2 case로 `evaluator.run`을 검증.
`_build_anchor_block` 와 `_testcases_excerpt` 헬퍼는 다음 edge cases 미커버:

- `_build_anchor_block(anchors=[])` — 빈 anchors → `"(no anchors loaded)"` 분기
- `_build_anchor_block` 다중 anchors — JSON serialization (한국어 ensure_ascii=False) / factors 없는 entry
- `_testcases_excerpt(testcases=[])` — 빈 list → `"(no testcases)"`
- `_testcases_excerpt` `kind` 없는 entry → `"?"` 표시
- `run` — `used_ids`가 anchor_id에 없는 경우 (LLM이 hallucinate) → 빈 list 반환

### 미뤄둔 이유
- 통합 테스트 happy path가 핵심 분기 cover.
- 단위 테스트 정비는 P9 audit B3 (architect/auditor coverage) 와 묶어 P12 polish 라운드에서 일괄.

### Resolution 시점
