## 4. 인사이트

### 4.1 IPE 의 detection 효과 (sample-level +9pp)

IPE 의 multi-stage verification (architect dual-call + algorithm_designer +
reviewer adversarial + executor sample run) 이 wrong sample 발견 능력 ↑.
baseline 이 자기 `expected` 를 mental-sim 오류로 생성한 것을 IPE 는 Phase A
sample mismatch 로 잡는다.

### 4.2 IPE 의 recovery 한계 (run-level -7pp)

발견은 하지만 budget 안에 fix 못 함. IPE r1 Two Sum sample 5/5 통과 + adv 5/5
통과인데 generator 단계에서 budget 소진. r3 BFS 도 sample 4/5 통과 후 generator
fail. **검증 layer 가 너무 strict 한 반면 recovery 메커니즘이 약함**.

### 4.3 **M3 dual-call 의 명백한 부작용**

Dijkstra:
- baseline: 3/3 (단순 알고리즘, LLM 이 매번 정확히 풀음)
- IPE: 0/3 (r3 architect 0-iter 로 problem 자체 못 만듦)

즉 **multi-mechanism 이 안정적 algorithm 까지 fail 시킴**. M3 dual-call 이
LIS / Dijkstra 처럼 잘 정의된 문제에서 두 모델 응답 disagreement → budget 빠르게
소진. **M3 의 net effect 가 음(-)** 가능성 강한 신호.

### 4.4 ECC subagent 패턴의 정당성 의문

ECC 의 가설: "Coder 책임 분리 → quality ↑". 측정 결과는 그 반대 — single-LLM
baseline 이 자기 풀 수 있는 algorithm 에 대해 더 안정적. **information bottleneck
hypothesis** (PRINCIPLES.md §1) 가 데이터로 부분 입증.

## 5. v0.3.0 release 판정

PRINCIPLES.md §3 의 결정 트리:

| 시나리오 | 판정 | 현재 데이터 |
|---|---|---|
| baseline ≫ IPE (Δ ≥ +20pp) | M1/M3/M4 rollback 검토 | — (Δ -7pp run, +9pp sample) |
| baseline ≪ IPE (Δ ≤ -20pp) | 현 방향 유효, budget tune | — |
| **baseline ≈ IPE** (\|Δ\| < 20pp) | **IPE 는 검증 layer 만 정당화** | **✓ 본 측정 (run -7pp / sample +9pp)** |

**판정**: **v0.3.0 tag 보류** + **multi-mechanism 일부 rollback 검토 PR**.

### 5.1 Rollback 후보 (우선순위)

1. **M3 dual-call rollback** (가장 명백한 음효과)
   - Dijkstra 3/3 vs 0/3 차이
   - r1 LIS / r2 LIS / r3 Dijkstra 의 architect 0-iter 패턴
   - rollback 시 single Opus call → IPE Phase 1 N=1 (Phase A 90% pass) 회복 가능
2. **M4 reviewer rollback 검토** (효과 측정 어려움)
   - reviewer reject 가 coder budget 빨리 소진하는 hint
   - 측정 데이터에서 명확한 음효과는 X
3. **M1 algorithm_designer 효과** (긍정 신호 없음)
   - Coder 분해 했지만 sample-level pass rate 향상이 검증 layer 덕분인지
     algorithm_designer 덕분인지 분리 안 됨
   - 후속 A/B 측정 필요

### 5.2 우선 행동

PRINCIPLES.md 룰 1 (N≥3) 와 룰 2 (cross-algorithm regression) 적용:
- M3 rollback PR 만들기 전에 단일 Opus architect call 의 5 algorithm × 3 run 측정
- IPE-without-M3 데이터가 IPE-with-M3 보다 좋으면 rollback 머지

## 7. 한계

- **15 runs / system 은 small sample size**. variance large — 동일 실험 재현 시
  점수 ±10pp variation 예상. 정확한 통계 결론은 N=10+ 필요.
- **Anthropic 529 overload 가 측정 중간에 한 번 발생** (IPE r3 1차 시도, 약 1
  시간 단절 후 회복). r3 재측정 데이터만 사용 — fail 1차 data 는 폐기.
- **Phase B/C 측정은 IPE 만의 가치 layer**. baseline 비교 영역 밖이라 그 부분의
  실제 value 는 본 측정으로 판단 X.
